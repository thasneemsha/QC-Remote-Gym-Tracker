# SR → FR → UC Traceability Matrix

This matrix maps Stakeholder Requirements (SR) to Functional Requirements (FR) and Use Cases (UC) for the QFit Universal Workout Tracker.

| Stakeholder Requirement (SR) | Functional Requirement (FR) | Mapped Use Case (UC) |
|------------------------------|-----------------------------|----------------------|
| **SR-1.1** Track workouts anywhere | FR-2.1 – FR-2.9 | UC-3 Log Workout |
| **SR-1.2** View progress charts | FR-3.1 – FR-3.3 | UC-4 View Progress |
| **SR-1.3** Set personal fitness goals | FR-4.1 – FR-4.4 | UC-5 Set Goals |
| **SR-1.4** Users shall be able to manage their account settings | FR-1.1 – FR-1.7 | UC-1 Register / Login |
| **SR-2.1** View users’ workout history | FR-5.1 – FR-5.3 | UC-6 Coach Review & Feedback |
| **SR-2.2** Analyze progress remotely | FR-5.3 – FR-5.5 | UC-6 Coach Review & Feedback |
| **SR-2.3** Provide feedback / recommendations | FR-5.4 – FR-5.10 | UC-6 Coach Review & Feedback |
| **SR-3.1** Manage users & roles | FR-6.1 – FR-6.4 | UC-7 Admin Management |
| **SR-4.1** Authenticate users | FR-1.4 – FR-1.5 | UC-1 Register / Login |
| **SR-4.2** Handle workout data | FR-2.1 – FR-2.9 | UC-3 Log Workout |
| **SR-4.3** Generate analytics | FR-3.2, FR-3.5 | UC-4 View Progress |
