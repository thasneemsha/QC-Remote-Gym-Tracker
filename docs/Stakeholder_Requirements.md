Stakeholder Requirements (SR)

The QFit Universal Workout Tracker is built to satisfy key stakeholder needs identified during the discovery phase. Below are the full stakeholder requirements categorized by user role:

---

### SR-1 User Requirements:
- **SR1.1** Track workouts anywhere  
- **SR1.2** View progress charts  
- **SR1.3** Set goals  
- **SR1.4** Users shall be able to update their personal profiles and settings  
- **SR1.5** Target user base (QC students) ensures technical familiarity and reduces system misusage  
- **SR1.6** System shall assume users possess baseline technical literacy, reducing onboarding complexity  

---

### SR-2 Coach Requirements:
- **SR2.1** View assigned usersâ€™ workout history  
- **SR2.2** Analyze progress remotely  
- **SR2.3** Provide feedback and recommendations  

---

### SR-3 Admin Requirements:
- **SR3.1** Manage users and roles  
- **SR3.2** Maintain system stability  
- **SR3.3** Ensure security and data integrity  

---

### SR-4 System Requirements:
- **SR4.1** Authenticate users  
- **SR4.2** Store, retrieve, and update workout data  
- **SR4.3** Generate analytics
